CSS Demo

A usable, highly responsive, WCAG accessible web form for any layout

Introduction

formReForm is a lightweight CSS framework designed to make web forms easier to style, more accessible, and responsive by default. It works with simple, semantic HTML and gives you control over layout without complex markup or JavaScript.

  • Built on modern CSS features like Flexbox and Grid.
  • Uses CSS variables so you can customize the look and feel from a single location.
  • Supports multiple label styles, responsive breakpoints, and flexible field arrangements.
  • Designed with accessibility in mind, including required indicators and screen-reader support.

Whether you’re building a simple contact form or a multi-section application, formReForm helps you keep forms clean, consistent, and user-friendly.

Usage & Syntax

formReForm uses a small set of structural classes to handle different layout modes. Each form element is wrapped in a .form-group and grouped horizontally or vertically with a .form-row .

Basic Structure


<div class="form-row">
  <div class="form-group">
    <label for="firstName">First Name</label>
    <input id="firstName" name="firstName" type="text" />
  </div>
  <div class="form-group">
    <label for="lastName">Last Name</label>
    <input id="lastName" name="lastName" type="text" />
  </div>
</div>

Layout Modes

  • Default (Stacked Labels) – Labels appear above their inputs.
  • Inline Labels – Add .form-inline to a .form-row for side-by-side labels and inputs with consistent label widths.
  • Checkbox Wrapping – Checkbox rows automatically wrap when they run out of space.
  • Checkbox Swap – Add .checkbox-swap to reverse the order of the checkbox and label.
  • Checkbox Columns – Add .checkbox-cols-3 , .checkbox-cols-4 , etc., to create evenly spaced checkbox grids.

Required Fields

Mark inputs with the required attribute to automatically add an asterisk (*) after the label. The indicator is customizable with the --required-character variable, and screen readers will read "required" based on the HTML attribute.

Tips & Hints

  • Inline Tips – Use .form-tip for small helper text under a field.
  • Popup Tips – Use .form-tip-popup for hover or focus-triggered tooltips without changing HTML structure.

Summary

The formReForm styles provide a flexible, accessible form system that adapts to different devices and layouts.

  • Use stacked labels by default or inline labels with consistent widths.
  • Automatically mark required fields with an asterisk. Screen readers announce required from the HTML attribute.
  • Add inline tips or popup hints without changing your HTML structure.
  • Top-align labels for taller inputs like textareas.
  • Lay out checkboxes with wrapping rows, swap the visual order, or display them in fixed columns.
  • Theme spacing, colors, and borders with CSS variables for easy reuse.

The goal is clean, responsive forms that stay consistent and easy to maintain.
